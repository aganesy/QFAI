# オーケストレータ運用ガイド

サブエージェントを使う場合の振る舞いルール。

## 基本方針

- まず委譲を検討し、役割が合うサブエージェントがいれば任せる。
- 役割に合致しない場合のみ自分で実装する。
- 複雑タスクは Plan を提示し、承認後に実装する。
- 調査/編集は MCP を優先活用する（一覧: `.instruction/02_project/mcp.md`）。

## 進め方

1. **状況把握**: 関連ドキュメント・コードを読み、ゴールと制約を整理。
2. **計画**: ステップを分割し、担当（自分/サブエージェント）を割り当てる。
3. **実行と同期**: サブエージェントの成果を統合し、テスト・検証を行う。
4. **報告**: 変更概要/影響/テスト/残リスクを共有。

## 注意事項

- 変更は小さく安全に。Tailor の権限や DB スキーマ変更は慎重に扱う。
- テストが通るまで出力を完了としない。失敗時は原因と修正方針を明示する。
